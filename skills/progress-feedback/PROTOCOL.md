# Progress Feedback Protocol

## 目标

实现长任务的实时进度反馈，避免用户等待焦虑。

---

## 规则

### 规则 1：立即确认（<3 秒）

**触发：** 收到任何消息

**操作：**
- 简单问题：直接回答
- 复杂任务：立即回复"收到，正在处理..."

**示例：**
```
✅ 收到！正在分析这个问题，预计需要 30-60 秒...
```

### 规则 2：定期更新（>30 秒任务）

**触发：** 任务执行超过 30 秒

**操作：**
- 每 30 秒推送一次进度
- 告知当前步骤和剩余时间

**示例：**
```
📊 进度更新：50%
当前：正在读取文件...
预计剩余：30 秒
```

### 规则 3：完成通知

**触发：** 任务完成

**操作：**
- 推送最终结果
- 总结执行时间和关键发现

**示例：**
```
✅ 完成！
执行时间：2 分 15 秒
关键发现：...
```

---

## 实现方式

### 方式 1：心跳机制

```powershell
$startTime = Get-Date
while ($task.Running) {
    $elapsed = (Get-Date) - $startTime
    if ($elapsed.TotalSeconds -gt 30) {
        Send-Progress "任务进行中... 已耗时 $($elapsed.TotalSeconds) 秒"
        $startTime = Get-Date  # 重置计时
    }
    Start-Sleep -Seconds 5
}
```

### 方式 2：工具调用钩子

在每个工具调用前后发送通知：
```
🔧 开始：读取文件...
🔧 完成：读取成功（2 秒）
🔧 开始：分析数据...
```

### 方式 3：预估时间

根据任务类型预估时间：
| 任务类型 | 预估时间 |
|---------|---------|
| 简单对话 | <5 秒 |
| 网络搜索 | 10-30 秒 |
| 文件读写 | 5-15 秒 |
| 多步骤任务 | 30-120 秒 |
| 复杂分析 | 60-300 秒 |

## 消息模板

### 开始模板
```
🎯 任务确认
内容：{task_description}
预计时间：{estimated_time}
状态：处理中...
```

### 进度模板
```
📊 进度更新
完成：{completed_percentage}%
当前：{current_step}
已用：{elapsed_time}
剩余：{remaining_time}
```

### 完成模板
```
✅ 任务完成
总耗时：{total_time}
结果：{result_summary}
下一步：{next_steps}
```
